# PlanCraft 사용자 매뉴얼

PlanCraft Agent를 사용하여 전문적인 비즈니스 기획서를 작성하는 방법을 안내합니다.

---

## 1. 시작하기 (Getting Started)

### 1.1 실행 방법
웹 브라우저에서 다음 주소로 접속합니다. (로컬 실행 시)
```
http://localhost:8501
```

### 1.2 화면 구성
1. **좌측 사이드바**: 품질 모드 선택, 이전 기록, 도움말
2. **메인 채팅창**: AI와 대화하며 기획서를 만드는 공간
3. **상태 표시줄**: 현재 어떤 에이전트가 작업 중인지 보여주는 타임라인

---

## 2. 기본 사용법 (Basic Usage)

### 2.1 기획서 작성 시작하기
채팅 입력창에 만들고 싶은 서비스 아이디어를 입력하세요.

- **짧고 단순하게**: "배달 앱 만들어줘"
  - AI 제안 모드: AI가 구체적인 컨셉(타겟, 차별점 등)을 제안하고, 사용자 확인을 거쳐 진행합니다.
- **구체적으로**: "직장인을 위한 점심 메뉴 추천 앱인데, 매일 11시에 알림이 오고, 팀원들과 투표할 수 있는 기능을 넣어줘."
  - Fast Track: AI가 바로 의도를 파악하고 기획서 작성을 시작합니다.

### 2.2 중간 확인 절차 (Human-in-the-Loop)
작업 도중 AI가 확실하지 않은 부분이 있으면 작업을 **일시 정지(Pause)**하고 질문합니다.
- 화면에 표시된 **옵션 버튼** 중 하나를 선택하거나,
- **직접 입력**을 통해 원하는 방향을 알려주세요.

---

## 3. 브레인스토밍 기능

아이디어가 없을 때 **브레인스토밍 버튼**을 클릭하면 AI가 트렌디한 서비스 아이디어를 제안합니다.

### 3.1 카테고리 선택
8개 카테고리 중 관심 분야를 선택하세요:

| 카테고리 | 설명 |
|----------|------|
| **IT/Tech** | 소프트웨어, SaaS, AI 서비스 |
| **금융/핀테크** | 결제, 투자, 자산관리 서비스 |
| **F&B** | 음식, 배달, 요식업 서비스 |
| **헬스케어** | 건강, 의료, 피트니스 서비스 |
| **교육** | 학습, 튜터링, EdTech 서비스 |
| **라이프스타일** | 일상, 취미, 반려동물 서비스 |
| **커머스** | 쇼핑, 구독, D2C 서비스 |
| **엔터테인먼트** | 콘텐츠, 게임, 미디어 서비스 |

### 3.2 사용 방법
1. 사이드바 또는 메인 화면에서 **브레인스토밍** 버튼 클릭
2. 카테고리 선택 (드롭다운)
3. AI가 해당 분야의 서비스 아이디어 3개 제안
4. 마음에 드는 아이디어 선택 → 기획서 작성 시작

## 4. 고급 기능 (Advanced Features)

### 4.1 품질 모드 설정 (Quality Presets)
사이드바에서 생성 모드를 선택할 수 있습니다.

| 모드 | 속도 | 특징 | 추천 상황 |
|------|------|------|----------|
| **Fast** | <1분 | 핵심 내용 위주, 시각자료 미포함 | 아이디어 스케치, 브레인스토밍 |
| **Balanced** | 2~3분 | 균형 잡힌 내용, Mermaid 다이어그램 포함 | 일반적인 기획서 작성 (기본값) |
| **Quality** | 3~5분 | 심층 분석, 복잡한 시각화, 3회 자체 검증 | 최종 보고서, 투자 제안서 |

### 4.2 시각적 요소
Balanced/Quality 모드에서는 기획서에 시각적 요소가 포함됩니다:

- **Mermaid 다이어그램**: 시스템 아키텍처, 사용자 플로우
- **ASCII 차트**: 성장 지표, 수익 예측 그래프

### 4.3 파일 업로드
기존에 작성한 메모나 참고 자료(PDF, TXT)가 있다면 업로드하세요.
AI가 해당 내용을 분석하여 기획서에 반영합니다.

### 4.4 제약조건 설정
특정 조건이 있다면 입력 시 함께 언급하세요:
- "광고 없이 구독 모델만 사용해줘"
- "React Native로 개발해야 해"
- "초기 투자금 5000만원 이내로"

AI가 모든 에이전트에게 해당 제약조건을 전달하여 일관성 있는 기획서를 생성합니다.

## 5. 자주 묻는 질문 (FAQ)

**Q. API 오류가 발생하면 어떻게 하나요?**
A. 일시적인 오류일 수 있으니 잠시 후 다시 시도해주세요. 개발자 도구의 "전체 테스트 실행" 기능을 통해 시스템 상태를 점검해볼 수 있습니다.

---

## 6. 개발자 도구 (Developer Tools)

PlanCraft는 개발자 및 고급 사용자를 위한 내장 도구를 제공합니다.

### 6.1 실행 방법
1. 화면 하단의 **"🛠 개발자 도구 (Dev)"** 버튼을 클릭합니다.
2. 모달 창이 열리며 다양한 탭을 확인할 수 있습니다.

### 6.2 주요 기능

#### ✅ Run ALL Tests (시스템 점검)
버튼 하나로 전체 시스템의 건강 상태를 진단할 수 있습니다.
- **기능**: 백엔드 로직, 에이전트 동작, DB 연결 등을 종합 테스트합니다.
- **결과 확인**: 테스트 완료 후 상세한 **HTML 리포트**가 화면에 표시됩니다. (Pass/Fail 여부 확인 가능)

#### 🧪 Agent Unit Test
특정 에이전트(Analyzer, Writer 등)만 독립적으로 실행하여 성능을 시험해볼 수 있습니다.

#### 📊 Workflow Graph
현재 실행 중인 에이전트 워크플로우를 다이어그램으로 시각화하여 흐름을 추적합니다.

## 7. 실시간 진행 상태 (Live Status)

기획서가 생성되는 동안, AI가 현재 무슨 생각을 하고 있는지 실시간으로 확인할 수 있습니다.
- **Thought Bubbles**: "🧐 요구사항 분석 중...", "🏗️ 목차 설계 중..." 과 같이 현재 작업 내용이 아이콘과 함께 표시됩니다.
- **투명성**: 단순히 기다리는 것이 아니라, AI의 작업 과정을 지켜볼 수 있어 지루하지 않습니다.

**Q. 실제 시장 데이터인가요?**
A. 네, Market Agent가 실시간 웹 검색(Tavily)을 통해 최신 시장 규모와 경쟁사 정보를 수집합니다. 단, AI의 특성상 일부 오차가 있을 수 있으므로 더블 체크를 권장합니다.

**Q. 일반적인 질문도 되나요?**
A. 네, "안녕하세요"나 "이 서비스의 장점이 뭐야?" 같은 가벼운 대화도 가능합니다. 이 경우 기획서는 생성되지 않고 챗봇처럼 답변합니다.

**Q. 다이어그램이 안 보여요.**
A. Fast 모드에서는 시각자료가 생성되지 않습니다. Balanced 또는 Quality 모드를 선택하세요.

**Q. 브레인스토밍 횟수 제한이 있나요?**
A. 세션당 10회까지 AI 브레인스토밍을 사용할 수 있습니다. 초과 시 정적 아이디어 풀에서 랜덤 제안됩니다.

## 6. 문제 해결 (Troubleshooting)

- **멈춤 현상**: 브라우저를 새로고침(F5) 하세요. 작업 내역은 남아있지 않을 수 있습니다.
- **에러 메시지**: "Max retries reached" 메시지가 뜨면, 입력 시간이 초과되어 기본값으로 진행된 것입니다. 결과를 확인 후 수정 요청하세요.
- **다이어그램 누락**: 품질 모드를 Balanced 이상으로 설정하고 다시 시도해보세요.
