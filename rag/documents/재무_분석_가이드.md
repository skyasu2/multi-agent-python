# 재무 분석 가이드

기획서의 "비즈니스 모델" 및 "재무 계획" 섹션 작성을 위한 핵심 개념과 계산 방법입니다.
구체적인 숫자와 논리적 근거가 있어야 신뢰도가 높은 기획서가 됩니다.

## 1. 핵심 재무 지표 (Unit Economics)

### 1.1 CAC (Customer Acquisition Cost) - 고객 획득 비용

#### 정의
한 명의 신규 고객을 획득하는 데 드는 비용

#### 계산식
```
CAC = 마케팅/영업 총비용 ÷ 신규 고객 수

예시:
- 마케팅비: 1,000만원/월
- 신규 가입자: 500명
- CAC = 1,000만원 ÷ 500명 = 20,000원
```

#### 채널별 CAC 벤치마크 (국내 기준)
| 채널 | CAC 범위 | 특징 |
|------|----------|------|
| 자연 유입 (SEO) | 0원 | 시간 소요, 장기 투자 |
| 바이럴/추천 | 1,000-5,000원 | 가장 효율적 |
| 인스타그램 광고 | 5,000-15,000원 | B2C 효과적 |
| 구글 검색 광고 | 10,000-30,000원 | 전환 의도 높음 |
| 네이버 광고 | 8,000-25,000원 | 국내 검색 트래픽 |
| 페이스북 광고 | 8,000-20,000원 | 타겟팅 정교 |
| 오프라인 영업 | 50,000-200,000원 | B2B 적합 |

### 1.2 LTV (Lifetime Value) - 고객 생애 가치

#### 정의
한 명의 고객이 서비스 이용 기간 동안 가져다주는 총 수익

#### 계산식 (구독 모델)
```
LTV = ARPU × 평균 이용 기간 × 마진율

예시:
- ARPU (월 객단가): 10,000원
- 평균 이용 기간: 12개월
- 마진율: 70%
- LTV = 10,000원 × 12개월 × 0.7 = 84,000원
```

#### 계산식 (거래 모델)
```
LTV = 평균 주문 금액 × 연간 구매 횟수 × 고객 유지 기간(년) × 마진율

예시:
- 평균 주문 금액: 50,000원
- 연간 구매 횟수: 4회
- 고객 유지 기간: 3년
- 마진율: 30%
- LTV = 50,000원 × 4회 × 3년 × 0.3 = 180,000원
```

### 1.3 LTV:CAC 비율

#### 건강한 비율 기준
| 비율 | 해석 | 판단 |
|------|------|------|
| 1:1 미만 | 고객 획득할수록 손해 | **위험** |
| 1:1 ~ 3:1 | 수익성 낮음 | 개선 필요 |
| **3:1 ~ 5:1** | **건강한 비즈니스** | **목표** |
| 5:1 초과 | 성장 여력 있음 | 마케팅 확대 가능 |

#### LTV:CAC 개선 방법
- **LTV 증가**: 객단가 인상, 이탈 감소, 업셀링
- **CAC 감소**: 바이럴 유도, 채널 최적화, 전환율 개선

## 2. 손익계산서 (P&L) 작성법

### 2.1 월별 손익 구조
```
매출 (Revenue)
├── 상품/서비스 매출
├── 구독 매출
└── 기타 매출

(-) 매출원가 (COGS)
├── 서버/인프라 비용
├── 결제 수수료
└── 콘텐츠/재료비
────────────────────
= 매출총이익 (Gross Profit)

(-) 판관비 (SG&A)
├── 인건비
├── 마케팅비
├── 임대료/사무비용
└── 기타 운영비
────────────────────
= 영업이익 (Operating Profit)
```

### 2.2 월별 손익 예시 템플릿
| 구분 | 1개월 | 3개월 | 6개월 | 12개월 |
|------|-------|-------|-------|--------|
| **매출** | | | | |
| └ 구독 매출 | 100만 | 300만 | 800만 | 2,000만 |
| └ 광고 매출 | 0원 | 50만 | 200만 | 500만 |
| **매출 합계** | 100만 | 350만 | 1,000만 | 2,500만 |
| **매출원가** | | | | |
| └ 서버 비용 | 30만 | 50만 | 100만 | 200만 |
| └ 결제 수수료 | 3만 | 10만 | 30만 | 75만 |
| **매출총이익** | 67만 | 290만 | 870만 | 2,225만 |
| **판관비** | | | | |
| └ 인건비 | 500만 | 500만 | 700만 | 1,000만 |
| └ 마케팅비 | 200만 | 300만 | 400만 | 500만 |
| └ 기타 | 100만 | 100만 | 150만 | 200만 |
| **영업이익** | -733만 | -610만 | -380만 | +525만 |

### 2.3 주요 마진율 기준
| 지표 | 계산식 | 목표 (SaaS) | 목표 (커머스) |
|------|--------|-------------|---------------|
| 매출총이익률 | 매출총이익 ÷ 매출 | 70-80% | 30-40% |
| 영업이익률 | 영업이익 ÷ 매출 | 10-20% | 5-10% |
| 순이익률 | 순이익 ÷ 매출 | 5-15% | 3-7% |

## 3. BEP (손익분기점) 분석

### 3.1 정의
총 비용과 총 매출이 같아지는 시점 (이익 = 0)

### 3.2 계산 방법

#### 고정비/변동비 분류
| 구분 | 특징 | 예시 |
|------|------|------|
| **고정비** | 매출과 무관하게 발생 | 인건비, 임대료, 서버 기본료 |
| **변동비** | 매출에 비례하여 발생 | 결제 수수료, 재료비, 배송비 |

#### BEP 계산식
```
BEP (매출 기준) = 고정비 ÷ (1 - 변동비율)

예시:
- 월 고정비: 1,000만원
- 변동비율: 30% (매출의 30%가 변동비)
- BEP = 1,000만원 ÷ (1 - 0.3) = 약 1,429만원/월
```

#### BEP (고객 수 기준)
```
BEP (고객 수) = 고정비 ÷ (객단가 - 고객당 변동비)

예시:
- 월 고정비: 1,000만원
- 월 구독료: 10,000원
- 고객당 변동비: 2,000원
- BEP = 1,000만원 ÷ (10,000 - 2,000) = 1,250명
```

## 4. 시나리오 분석 (Scenario Planning)

### 4.1 3가지 시나리오
| 시나리오 | 가정 | 용도 |
|----------|------|------|
| **Best Case** | 모든 것이 잘 풀림 | 상한선 파악 |
| **Base Case** | 현실적인 예상 | 기본 계획 |
| **Worst Case** | 주요 리스크 발현 | 최소 생존선 |

### 4.2 시나리오별 변수 설정 예시
| 변수 | Worst | Base | Best |
|------|-------|------|------|
| 월간 신규 가입 | 100명 | 300명 | 500명 |
| 유료 전환율 | 2% | 5% | 8% |
| 이탈률 | 15% | 8% | 5% |
| CAC | 30,000원 | 20,000원 | 15,000원 |

### 4.3 시나리오별 12개월 결과 예시
| 지표 | Worst | Base | Best |
|------|-------|------|------|
| 누적 사용자 | 800명 | 3,000명 | 5,500명 |

## 5. 투자 유치 시 재무 관련 FAQ

### Q1: 밸류에이션은 어떻게 산정하나요?
#### 초기 스타트업 방법
- **MRR 배수**: MRR × 80-150배 (SaaS)
- **ARR 배수**: ARR × 8-15배
- **GMV 배수**: GMV × 0.5-2배 (플랫폼)

### Q2: Burn Rate란?
#### 정의
월간 순 현금 유출액 (= 비용 - 매출)

#### 계산
```
Burn Rate = 월간 지출 - 월간 수입
Runway = 보유 현금 ÷ Burn Rate

예시:
- 보유 현금: 1억원
- Burn Rate: 2,000만원/월
- Runway = 1억 ÷ 2,000만 = 5개월
```

### Q3: Runway는 얼마나 확보해야 하나요?
| 상황 | 권장 Runway |
|------|-------------|
| 시드 단계 | 12-18개월 |
| 시리즈 A | 18-24개월 |
| 스케일업 | 12개월 이상 |
