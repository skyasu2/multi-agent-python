# 경쟁사 분석 프레임워크

기획서의 "경쟁사 분석" 및 "차별화 전략" 섹션 작성을 위한 체계적인 분석 방법론입니다.
최소 3개 이상의 경쟁사를 분석하고, 각각에 대한 우리의 차별점을 명시해야 합니다.

---

## 1. 경쟁사의 유형

### 1.1 경쟁사 분류
| 유형 | 정의 | 예시 (배달앱 기준) |
|------|------|-------------------|
| **직접 경쟁자** | 동일한 문제를 동일한 방식으로 해결 | 배달의민족, 쿠팡이츠 |
| **간접 경쟁자** | 동일한 문제를 다른 방식으로 해결 | 직접 방문 포장, 밀키트 |
| **잠재 경쟁자** | 진입 가능성이 있는 플레이어 | 네이버, 카카오 |
| **대체재** | 근본적으로 다르지만 대체 가능 | 집에서 요리, 편의점 |

### 1.2 경쟁사 맵핑

## 4. 차별화 전략

### 4.1 차별화 유형
| 유형 | 설명 | 예시 |
|------|------|------|
| **제품 차별화** | 기능/성능/디자인 우위 | 애플 (디자인), 다이슨 (기술) |
| **가격 차별화** | 원가 우위/가격 경쟁력 | 샤오미, 이케아 |
| **서비스 차별화** | 고객 경험/지원 우위 | 자포스 (고객서비스) |
| **채널 차별화** | 유통/접근성 우위 | 테슬라 (직접 판매) |
| **브랜드 차별화** | 이미지/신뢰 우위 | 나이키, 스타벅스 |
| **니치 차별화** | 특정 세그먼트 집중 | 롤스로이스 (초고급) |

### 4.2 경쟁 우위 원천 (Moat)

#### 진입 장벽 유형
| 해자 유형 | 설명 | 예시 |
|-----------|------|------|
| **네트워크 효과** | 사용자가 많을수록 가치 증가 | 카카오톡, 링크드인 |
| **전환 비용** | 이동 시 비용/불편 발생 | 엑셀 (학습 비용), AWS (마이그레이션) |
| **데이터 해자** | 축적된 데이터가 경쟁 우위 | 구글 (검색), 넷플릭스 (추천) |
| **브랜드** | 인지도/신뢰도 우위 | 코카콜라, 삼성 |
| **규모의 경제** | 규모가 클수록 원가 절감 | 아마존, 쿠팡 |
| **특허/규제** | 법적 독점권 | 제약사, 퀄컴 |

### 4.3 블루오션 전략 (ERRC)
| 전략 | 질문 | 행동 |
|------|------|------|
| **Eliminate** | 업계가 당연시하지만 없앨 것? | 제거 |
| **Reduce** | 업계 수준 이하로 줄일 것? | 축소 |
| **Raise** | 업계 수준 이상으로 높일 것? | 강화 |
| **Create** | 업계에 없는 새로운 것? | 창조 |

#### ERRC 예시: 시르크 드 솔레이유
| 전략 | 기존 서커스 | 시르크 드 솔레이유 |
|------|------------|-------------------|
| Eliminate | 동물 공연 | 제거 |
| Reduce | 스릴/위험 | 축소 |
| Raise | 예술성 | 강화 |
| Create | 테마/스토리 | 창조 |

## 6. 경쟁사 분석 체크리스트

### 기획서 제출 전 확인사항
- [ ] 최소 3개 이상 경쟁사 분석
- [ ] 직접 경쟁사 + 간접 경쟁사 포함
- [ ] 각 경쟁사별 강점/약점 명시
- [ ] **각 경쟁사별 "우리의 차별점" 필수** (가장 중요!)
- [ ] 기능 비교 테이블 포함
- [ ] 차별화 전략 명확히 기술
- [ ] 경쟁 우위 지속 가능성 설명

### 흔한 실수
- [ ] 경쟁사가 "없다"고 주장 (투자자가 가장 싫어하는 답변)
- [ ] 경쟁사 약점만 나열, 강점 무시
- [ ] "우리가 더 좋다"는 주장만 있고 근거 없음
- [ ] 차별점이 모호하거나 지속 불가능
